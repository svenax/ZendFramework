Criando e Carregando Documentos PDF

A classe Zend_Pdf representa os documentos PDF e provê funcionalidades para manipulação do documento.

Para criar um novo documento, um novo objeto Zend_Pdf deve ser instanciado primeiro.

A classe Zend_Pdf também provê dois métodos estáticos para carregar um documento PDF existente. Os métodos são Zend_Pdf::load() e Zend_Pdf::parse(). Ambos retornam objetos Zend_Pdf como resultado, ou uma exceção se algum erro ocorrer.

Example #1 Criar um novo documento PDF ou carregar um já existente

  1. ...
  2. // Cria um novo documento PDF
  3. $pdf1 = new Zend_Pdf();
  4.  
  5. // Carrega um documento PDF a partir de um arquivo
  6. $pdf2 = Zend_Pdf::load($fileName);
  7.  
  8. // Carrega um documento PDF a partir de uma string
  9. $pdf3 = Zend_Pdf::parse($pdfString);
  10. ...

O formato de arquivo PDF suporta a atualização incremental do documento. Dessa forma, toda vez que um documento é atualizado, uma nova revisão do documento é criada. O componente Zend_Pdf suporta a recuperação de uma revisão especificada.

Uma revisão pode ser especificada como o segundo parâmetro para os métodos Zend_Pdf::load() e Zend_Pdf::parse() ou solicitada chamando o método Zend_Pdf::rollback(). [1] Zend_Pdf::rollback()

Example #2 Solicitando Revisões Específicas de um Documento PDF

  1. ...
  2. // Carrega a revisão anterior do documento PDF
  3. $pdf1 = Zend_Pdf::load($fileName, 1);
  4.  
  5. // Carrega a revisão anterior do documento PDF
  6. $pdf2 = Zend_Pdf::parse($pdfString, 1);
  7.  
  8. // Carrega a primeira revisão do documento PDF
  9. $pdf3 = Zend_Pdf::load($fileName);
  10. $revisions = $pdf3->revisions();
  11. $pdf3->rollback($revisions - 1);
  12. ...
[1] O método deve ser chamado antes que qualquer mudança seja aplicada ao documento, caso contrário o comportamento do método não é definido.
